Introduction

Git

In the world of version control system, Git is known to be the most widely used version control system. Git is an open-source project originally developed in 2005 by linus Trovalds to help in the working of Linux kernel.

Version Control System

Since git is a version control system, let's talk briefly on version control system (VCS). VCS is a system that keeps record of every changes of a file or a set of files over time so you can always recall them later. If you wreck your project while working on it, by any chance (which you will likely to), you can always restore to previous versions and continue work from there

Distributed VCS

Even among the version control system, Git is of distributed architecture. In this architecture, there is no single centralized repository. A cloned project in each contributor's local machince can serve as a repository.
